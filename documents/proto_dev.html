1 协议说明

1.1 字段说明

  • 必要字段名以"*"标识,
  • 非业务字段以"_"开始
  • 数据发送 json -> msgpack(压缩), 数据接收 msgpack(解压) -> json

1.2 Http说明

1.2.1 http header

  • Api-Vern 接口版本，默认1.0.0
  • Content-Type application/json

1.3 设备指令

1.3.1 通用功能 0x0001-0x00FF

  • 升级通知 0x0001
  • 启动 0x0002
  • 停止 0x0003
  • 重启 0x0004

1.3.2 充电设备 0x0101-0x01FF

  • 充电 0x0101
  • 断电 0x0102

1.4 rspCode

  • 0 成功

2 协议定义

2.1 开机 http post /dev/init/start

设备启动时通知服务器

  • req 
{ 
    id*: 设备唯一标识,
    fmwManu*: 设备厂商 如YYDZ-友友电桩 TODO 是否需要上传
    fmwModl*: 固件类型 如cc_c,cc_i,cc_o TODO 是否需要上传
    fmwVern*: 固件版本 如1_0_0
}
  • rsp 
{
    rspCode*: 响应码 0-成功
    rspDesc: 响应描述
    mq: {
        uri: mq地址 TODO 集群
        subTopic: [/dev/ota/fmw_no,/dev/ins/clientId] #订阅主题 ota-升级通知 ins-设备指令
        clientId: mq的客户端标识
    }
    ping: 60  心跳间隔时间，单位秒
}

2.2 关机 http post /dev/init/stop

设备关闭时通知服务器

  • req 
{ 
    id*: 设备唯一标识
}
  • rsp 
{
    rspCode*: 响应码 00-成功
}

2.3 OTA升级通知 mtqq

服务器通知新固件可升级，设备先判断指令，再依据url验证是否符合自己的固件

  • s2c 
{
    url*: 升级地址
    ins: 0x0001
}

2.4 获取OTA升级地址 http post /dev/ota/upgurl 废弃不用

设备知道可以升级后，获取下载地址

  • req 
{ 
    _v: 接口版本, 默认1_0_0
    id*: 设备唯一标识
}
  • rsp 
{
    rspCode*: 响应码 00-成功 TODO 异常处理和重试机制
    url*: 升级地址 如http://ota.ev.com/YYDZ_cc_c_1_2_0.gz
}

2.5 开始充电 mqtt

服务器通知设备开始充电

  • s2c 
{   
    ins*: 0x0101
    id*:  插座ID
}

2.6 停止充电 http post /dev/charge/stop

设备通知服务器停止充电

  • req 
{ 
    _v: 接口版本, 默认1_0_0
    id*: 插座ID
}
  • rsp 
{
    rspCode*: 响应码 00-成功
}

2.7 停止充电 mqtt

服务器通知设备停止充电

  • s2c 
{ 
    ins: 0x0102
    id*:  插座ID
}

2.8 异常 http post /dev/info/err

设备通知异常给服务器

  • req 
{ 
    id*: 设备ID
    errCode*: 错误码
    errDesc: 错误描述
}
  • rsp 
{
    rspCode*: 响应码 0-成功
}

2.9 充电度数 http post /dev/charge/degree

设备上传充电度数

  • req 
{ 
    id*: 插座ID
    degree*: 充电度数
}
  • rsp 
{
    rspCode*: 响应码 0-成功
}

2.10 设备心跳 http post /dev/io/ping

设备心跳,间隔时间由start获取

  • req 
{ 
    id*: 设备ID
}
  • rsp 
{
    rspCode*: 响应码 0-成功
}
